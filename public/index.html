How it works

Under the hood

A clean, four-layer architecture — each layer does one thing well.

1

Upload

Admin portal accepts PDF, DOCX, XLSX. pdfplumber, python-docx, and openpyxl extract raw text.

2

Generate

Model reads the text in chunks and produces Q&A pairs stored in MongoDB with your user_id.

3

Serve

FastAPI serves Q&A over REST. The chat endpoint filters by user_id and returns the best Gemini answer.

4

Embed

A vanilla JS widget with Shadow DOM renders a floating chat bubble. One script tag, zero dependencies.

One script tag. That's it.

Drop this anywhere in your HTML and a live FAQ chatbot instantly appears on your site.

<!-- AQ Chatbot Widget --> <script src="https://your-domain.com/widget.js" data-api="https://your-api.com" data-user-id="your-user-id" ></script>
